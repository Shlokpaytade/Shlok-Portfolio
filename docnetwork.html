Day 1: Introduction to Networking

Building blocks of networking include sender, receiver, transmission medium (cables, wireless), message, and protocol.

Modes of communication are simplex, half-duplex, and full-duplex - for example, walkie-talkies vs. video calls.

Understanding analog vs digital signals, with digital dominant due to noise immunity and ease of processing.

Essential characteristics of networking: speed, accuracy, reliability, and security.

Real-world example illustrating WhatsApp packet journey across networks.

Day 2: Types of Network & Nature of Data

  • Network types: LAN, MAN, WAN, PAN – spanning home Wi-Fi to global ISPs.
  • Comparison of Client-server vs. peer-to-peer networks and their effect on scalability.
  • Bandwidth and latency trade-offs, important in scenarios like high-frequency trading needing low latency.
  • Types of data: oriented (structured), observed (sensor), inferred (analytics).
  • Insights into IoT where multiple network types coexist.

Day 3: The OSI Model Demystified

Understanding the OSI 7-layer model: Physical, Data Link, Network, Transport, Session, Presentation, Application.

Examples of protocols operating at each layer: Ethernet, IP, TCP/UDP, SSL/TLS, HTTP/HTTPS, FTP.

Concepts of encapsulation & de-encapsulation: how headers/footers are applied at each layer.

Hands-on: tracing a packet using ping and traceroute.

Day 4: Inside the Web - HTTP & DNS

DNS converts domain names to IP addresses functioning as the internet's "phonebook".

HTTP/HTTPS request methods include GET, POST, PUT, DELETE - used in RESTful APIs.

Caching mechanisms and HTTP response codes: 2xx (success), 3xx (redirection), 4xx (client error), 5xx (server error) to manage efficient web traffic.

Lab: using nslookup and browser developer tools to observe DNS lookups live.

Day 5: Revision & DHCP in Action

Quick revision of OSI model concepts.

Automatic IP management using DHCP (Dynamic Host Configuration Protocol).

Differences between static and dynamic IP addresses.

DHCP handshake steps: Discover → Offer → Request → Acknowledge.

Practical: viewing router’s DHCP lease table and observing dynamic IP address assignments.

Day 6: Low-Level Protocols & Diagnostics

  • Role of underlying protocols in network reliability.
  • ARP: Address Resolution Protocol maps IP to MAC addresses within LANs.
  • ICMP: Internet Control Message Protocol powers tools like ping and traceroute.
  • DHCP Acknowledge: confirms IP address assignment.

Practical: tracing packets and locating network bottlenecks using these protocols.

Day 7: From Theory to Zynq-Based App

Integrating learning into a mini project.

Designed and submitted a Zynq FPGA-based application using DHCP, ARP, and ICMP protocols.

Understanding embedded-to-network communication.

Transferable skills gained for IoT, industrial automation, and edge computing applications.
