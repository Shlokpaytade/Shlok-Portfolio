DAY-3

On 27 March, day-3 of Digital FABLAB, we focused on practical applications of HTML tags we learned previously. We had hands-on practice with various tags, with special emphasis on the img tag. We learned how to:

  • Insert images using the img tag with src, alt, width, and height attributes
  • Create responsive images
  • Position images within the page
  • Create image galleries
  • Use figure and figcaption tags

We also practiced creating forms with input fields, select dropdowns, and buttons. The practical session helped reinforce our understanding of HTML structure and elements.

DAY-4

On 28 March, day-4 of Digital FABLAB, we were introduced to the basics of CSS (Cascading Style Sheets). Our mentor explained the three ways to include CSS in HTML:

  1. Inline CSS - using the style attribute
  2. Internal CSS - using the style tag in the head section
  3. External CSS - linking a separate .css file

We learned basic CSS properties such as color, background-color, font-size, margin, padding, and border. We practiced applying CSS to HTML elements to improve the design and layout of web pages.

DAY-5

On 29 March, day-5 of Digital FABLAB, we explored CSS selectors, the box model, and positioning elements. Our mentor introduced us to class and ID selectors, pseudo-classes, and combinators. We also studied how margin, padding, border, and content form the CSS box model and practiced positioning elements using static, relative, absolute, and fixed positions.

DAY-6

On 30 March, day-6 of Digital FABLAB, we started learning about JavaScript, the programming language of the web. JavaScript helps make web pages interactive and dynamic beyond static HTML and CSS.

The topics covered included:

  1. Introduction to JavaScript syntax and variables
  2. Data types: strings, numbers, booleans, arrays, and objects
  3. Basic operators and expressions
  4. Functions: declaration, calling, and scope
  5. Working with the console for debugging
  6. JavaScript integration in HTML using <script> tags

DAY-7

On 31 March, day-7, we explored more advanced JavaScript features, focusing on string handling and format matching. These concepts are essential for validating user input and formatting data dynamically.

Key learnings were:

  • Using regular expressions (regex) to match patterns within strings.
  • The test() method to check for matches.
  • Using string methods like match(), replace(), and search().
  • Examples of formats: email validation, phone number patterns.
  • Practical practice of creating regex patterns for format matching.
